## Overview

This portfolio is more than a presentation layer — it's a proof of concept.

It demonstrates that the boundary between "technical" and "non-technical" work is dissolving, and that professionals who understand **systems thinking, clear communication, and AI tooling** can build production-grade software without writing code from scratch.

I built this site using a methodology I call **Vibe Coding**: treating AI agents as a junior development team, and positioning myself as the product manager, architect, and QA lead.

## The Constraint That Forced Creativity

I needed a high-performance portfolio that could compete visually and technically with developer-built sites. The standard options — Wix, Squarespace, Webflow — trade flexibility for ease. I wanted neither the limitations nor the generic aesthetic.

The constraint: **no deep coding knowledge, high standards, zero tolerance for drag-and-drop limitations.**

The solution: reframe the problem entirely.

## The Vibe Coding Methodology

Instead of learning React and TypeScript from scratch, I structured the development process as an **AI-assisted product workflow**:

**1. Foundation Selection**
I chose Magic Portfolio — a high-quality open-source template built on Next.js — as the architectural base. Starting with a solid foundation meant AI agents were modifying existing patterns rather than generating from nothing, which dramatically reduced error rates.

**2. Prompt Engineering as Project Management**
Every modification was driven by precise, context-rich prompts. Vague instructions produce vague code. I learned to communicate system requirements the way a product manager communicates to engineers: specific inputs, expected outputs, and clear success criteria.

**3. Iterative Debugging Loops**
When errors occurred, I fed the complete error logs back to the AI with context about what I was trying to achieve. This created a debugging loop that resolved issues faster than traditional Stack Overflow research — and taught me the underlying logic of the system in the process.

**4. Continuous Deployment**
The site runs on **Vercel**, connected directly to GitHub. Every approved change deploys automatically — no manual upload, no FTP, no build commands. This CI/CD pipeline means updates go live in under two minutes.

## What Was Actually Built

This isn't a template with swapped text. The customizations include:

- Custom routing and content architecture using MDX
- Modified component layouts for work experience, certifications, and skills sections
- Custom gallery logic supporting both images and YouTube video embeds with dynamic aspect ratio handling
- SEO metadata and Open Graph image generation
- Password-protected routes for selective content visibility
- Integrated newsletter and social sharing infrastructure

## The Broader Point

This project is evidence of a shift in what technical competency means.

Syntax is increasingly abstracted away. What remains — and what becomes more valuable — is the ability to **define problems precisely, design systems logically, and communicate requirements clearly enough for an AI to execute them.**

That's not a lesser form of technical work. It's the direction technical work is heading.

- **Development Method:** AI-Assisted (Vibe Coding)
