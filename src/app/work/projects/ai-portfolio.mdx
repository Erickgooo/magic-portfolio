## Overview

This portfolio is itself a case study.
It demonstrates something I believe is increasingly true: the boundary between "technical" and "non-technical" work is dissolving. Professionals who combine systems thinking, precise communication, and AI tooling can now build production-grade software without writing code from scratch — and the output is indistinguishable from developer-built work.
This site is evidence of that.

## The Constraint

I needed a portfolio that could compete visually and technically with sites built by engineers. The standard options — Wix, Squarespace, Webflow — trade real flexibility for ease of use. I wanted neither the limitations nor the generic aesthetic that comes with them.
The constraint: no deep coding background, high visual and technical standards, and zero tolerance for drag-and-drop ceilings.
The solution: reframe what "building" means.

## The Methodology — AI as a Development Team

Rather than learning React and TypeScript from scratch, I structured the process as an AI-assisted product workflow — positioning myself as the product manager, architect, and QA lead, and treating AI agents as the execution layer.

### 1. Foundation Selection
I started with Magic Portfolio — a production-grade open-source template built on Next.js. Beginning with a solid architectural base meant AI agents were modifying existing, well-structured patterns rather than generating from nothing. This dramatically reduced error rates and kept the codebase coherent.

### 2. Prompt Engineering as Specification Writing
Every modification was driven by precise, context-rich prompts. Vague instructions produce vague code. I learned to communicate requirements the way a product manager communicates to engineers: specific inputs, defined outputs, and clear success criteria. The quality of the prompt determined the quality of the result.

### 3. Iterative Debugging
When errors occurred, I fed complete error logs back to the AI with full context about the intended behavior. This created a tight debugging loop that resolved issues faster than traditional research — and in the process, taught me the underlying logic of the system I was building.

### 4. Continuous Deployment
The site runs on Vercel, connected directly to a GitHub repository. Every approved change deploys automatically — no manual uploads, no build commands. Updates go live in under two minutes. The domain erickmahecha.com is custom-purchased and configured end-to-end.

## What Was Built

This isn't a template with swapped text and placeholder images. The build includes custom routing and content architecture, modified component layouts across multiple sections, gallery logic that handles both images and embedded YouTube video with dynamic aspect ratio, SEO metadata and Open Graph image generation, password-protected routes for selective content visibility, and integrated newsletter infrastructure.

**The stack:** Next.js 15 + TypeScript · Once UI · MDX · Vercel + GitHub CI/CD

**AI tools used in development:** Google Gemini 3 Pro · Claude

## The Broader Point

Syntax is increasingly abstracted away. What becomes more valuable in its place — and what this project required — is the ability to define problems precisely, design systems logically, and communicate requirements clearly enough for an AI to execute them reliably.
That's not a lesser form of technical work. It's the direction technical work is heading.
